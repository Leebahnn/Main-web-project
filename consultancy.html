1.4 The different risks that might impact your project :


Once your project is launched, you need to think of it as an ongoing project that represents the image of you business.
So it is important to analyse and understand the different risks at the creation of your project to plan ahead of the problematics you may encounter, and so that you will have a finished product that serves you in the long term. As any other web based projects it needs maintenance to keep your site upto to date, and the best way to do that is to have someone to monitor your project from the launch and to it's entirety, to identify if there are other risks.


1.5 Selecting and using IT systems and software


At first we need to know the scale of the project we are working on, so that we can choose the right tech stack.
A tech stack is the combination of programming languages, tools and frameworks, which is then divided in two software components called front-end(client-side) and back-end(server-side).
When we know the scale of our project whether it's a small one,like an artist's website to put his work out and where it needs to be delivered as soon as possible we can use Wordpress.
Or a big project, like an e-commerce or a databased website where we need a mixture of programming languages(Javascript,CSS,Python,HTML) and web apps (Java,C#).
We should also see how big the community is behind the language or framework and see if the interest of the software is still growing. Because there is no point of choosing something old if there is something new better, faster and more responsive.
But your projects always determines choosing the right stack as we can easily get lost with all the different tools available in today's world.
